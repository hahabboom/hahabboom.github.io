第二章 物理层

1. 概念：

考虑的是怎样才能在连接各种计算机的传输媒体上传输数据比特流，而不是指具体的传输媒体

2. 主要任务：

确定与传输媒体的接口有关的一些特性，即：

  1. 机械特性 —— 指明接口所用接线器的形状和尺寸、引脚数目和排列、固定和锁定装置

  2. 电气特性 —— 指明在接口电缆的各条线上出现的电压的范围

  3. 功能特性 —— 指明某条线出现的某一电平的电压的意义

  4. 过程特性 —— 指明对于不同功能的各种可能时间的出现顺序

常用的物理层接口标准有 EIA RS-232-C、ADSL、 SONET/SDH

4. 编码（转换为数字信号）

1. 数字数据 -> 数字信号

- 归零编码 —— 正脉冲（1），负脉冲（0）
    归零占据一部分带宽，传输效率受到影响
- 非归零编码 —— 正电平（1），负电平（0）
    跟上面的区别是不用归零，一个周期取来传输数据，但无法传递时钟信号，双方难以同步
- 反向非归零 —— 信号翻转（0），保持不变（1）
    翻转的信号本身可以作为一种通知机制 === USB2.0
- 曼彻斯特编码 —— 周期中心向上跳变（0），周期中心向下（1）
    但他所占的频带宽度是原始基带宽度的两倍
- 差分曼彻斯特编码 —— 中心都有跳变，位开始边界有边边（0），无跳变（1）
    码元中间有一次跳变，可以实现跳变，抗干扰性较好 === 局域网
- 4B/5B编码 —— 将4位转化为对应的5位码

2. 模拟数据 -> 数字信号

采用对音频信号进行编码的脉码调制（PCM），包括采样、量化和编码

采样频率 >= 2 * 最大频率

10. 物理层设备

####1. 中继器

功能：将信号整形并放大在转发出去
原理：信号再生
有两个端口，端口仅作用于信号的电气部分，不管数据中是否有错误数据或不适于网段的数据；两端是相同的媒体；两端的网络是网段不是子网
原理上，中继器的使用数目是无限的，网络因而也可以无限延长；但不可能，有5-4-3规则

####2. 集线器（其实是多端口的中继器）

主要使用双绞线组建共享网络

11. 数据交换方式

####1. 电路交换

阶段：连接建立、数据传输、连接释放

优点：
1. 通信时延小
2. 有序传输
3. 没有冲突
4. 适宜范围广，即模拟信号和数字信号都能传输
5. 实时性强，适用于交互式会话类通信
6. 控制简单
缺点：
1. 建立连接时间长
2. 线路独占
3. 灵活性差，即通路中出现一点故障，就必须重新建立新的连接，对于十分紧急和主要的通信是很不利的
4. 难以规格化，即数据直达，不同类型、规格、速率的终端很难相互通信，难以进行差错控制
5. 没有存储能力，难以平滑通信量

适宜：要传送的数据量很大且传送时间远大于呼叫时间时，采用电路交换

####2. 报文交换（存储转发，利用路由信息着找一个节点）

优点：
1. 无需建立连接
2. 动态分配线路
3. 提高线路可靠性
4. 提高线路利用率
5. 提供多目标服务
6. 在存储转发中容易出现代码转换和速率匹配，甚至收发双方可以不同时处于可用状态。便于类型、规格和速度不同的计算机之间进行通信
缺点：
1. 实时性差，会引起转发时延
2. 只适用于数字信号
3. 报文交换对报文大小没有限制，这要求网络节点需要较大缓存空间

####3. 分组交换（存储转发，利用路由信息着找一个节点）

同报文交换一样，但解决了报文交换中大报文传输的问题；限制了每次传送的数据块大小的上限

优点：
1. 无建立时延
2. 线路利用率高
3. 简化了存储管理（相对于报文交换）
4. 加速传输，即传输一个分组所需的缓冲区比一个报文的小得多，这样因缓冲区不足而等待发送的概率及时间也少很多
5. 减少了出错概率和重发数据量
6. 分组短小，适用于计算机之间突发式数据通信
缺点：
1. 存在传输时延（但还是比报文交换的时延小，相对于电路交换存在存储转发时延）
2. 需要额外的信息量（每个小数据块都要加上源地址、目的地址和分组编号等信息，使得传送的信息量增大了5%~10%）
3. 当分组交换采用数据报服务时，可能会出现失序、丢失或重复分组，分组到达目的结点，要对分组编号进行排序等工作

适宜：端到端的通路由多段链路组成时，尤其适合突发式数据通信

1. 数据报方式（网络层提供）

特点：
1. 发送方可随时发送分组，网络中的节点可随时接收分组
2. 网络尽最大能力交付，传输不保证可靠性，所以可能丢失；为每个分组独立地选择路由，转发的路径可能不同，因而分组不一定按序到达
3. 发送的分组中要包括发送端和接收端的完整地址，以便可以独立传输
4. 分组在交换节点存储转发时，需要排队等候处理，会带来一定时延。
5. 网络具有冗余路径，当某一交换节点或一段链路出现故障，可相应地更新转发表，寻找另一条路径转发分组，对故障适应能力强
6. 存储转发的时延一般较小，提高网络的吞吐量
7. 收发双方不独占某一链路，资源利用率高

2. 虚电路方式（网络层提供）

本条电路不是专用

在传送数据时，每个数据分组不仅要有分组号、校验和等控制信息，还要有虚电路号

特点：
1. 虚电路通信链路的建立和拆除需要时间开销，对交互式应用和小量的短分组情况显得很浪费，但对长时间、频繁的数据交换效率高
2. 虚电路的路由选择体现在连接建立阶段，连接建立后，就确定传输路径
3. 虚电路提供可靠的通信功能，能保证每个分组正确且有序到达。同时还可以进行流量控制
4. 当网络中的某个结点或某条链路出现故障而彻底失效时，所有经过该结点或该链路的虚电路遭到破坏
5. 分组首部不包含目的地址，而包含虚电路标识符，相对数据报方式开销小

3.数据报服务和虚电路服务比较
数据报服务 虚电路服务
连接的建立 不需要 必须要
目的地址 每个分组都有完整的地址 仅在建立连接的阶段使用
路由选择 每个分组独立进行路由选择和转发 属于同一条虚电路的分组按同一路由转发
分组顺序 不保证顺序到达 保证顺序到达
可靠性 不保证可靠通信，可靠性由用户主机保证 可靠性由网络保证
对网络故障的适应性 出现故障的节点丢失分组，其他分组路径选择发生变化可以正常传输 所有经过故障结点的虚电路都不能正常工作
差错处理和流量控制 由用户主机控制，不保证数据报的可靠性 可由分组交换网络负责，也可由用户主机负责
