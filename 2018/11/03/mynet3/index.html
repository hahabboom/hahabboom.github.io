计算机网络自我总结数据链路层
发表于 2018-11-03| 更新于 2020-12-28|计算机网络
阅读量:

1. 功能：

在物理层提供服务的基础上向网络层提供服务，其主要作用是加强物理层传输原始比特流的功能，

将物理层提供的可能出错的物理连接改造为逻辑上无差错的数据链路，使之对网络层表现为一条无差错的链路。

1. 为网络层提供服务
    - 无确认的无连接服务（实时通信或误码率较低的通信信道，以太网）
    - 有确认的无连接服务（误码率较高的通信信道，无线通信）
    - 有确认的面向连接服务（通信要求（可靠性，实时性）较高的场合）
2. 链路管理
3. 帧定界、帧同步和透明传输
    - 帧同步：指接收方能够从接受到的二进制比特流中区分出帧的起始于终止
    - 帧定界： 确定帧的界限
    - 透明传输： 不管所传的数据是什么样的比特组合，偶读能够在链路上传输
4. 流量控制
    - 限制发送方的数据流量
    - 流量控制不是数据链路层特有的功能
    - 数据链路层控制的是相邻两结点之间数据链路上的流量，而对于传输层控制的是从源端到目的端之间的流量
5. 差错控制（包括位错 + 帧错）
    - 位错 —— 采用循环冗余码（CRC）来发现位错，通过自动重传（ARQ）方式来重传出错的帧（发现错了就丢弃，发送方就收不到确认，超时就会再发）
    - 帧错 —— 帧丢失，帧重复，帧失序

2. 数据链路层基本问题：封装成帧、透明传输、差错检测

1. 组帧

1. 字符计数法（计数字段包含自身所占用的字节）
2. 字符填充法
3. 零比特填充（每五个1，在后面就加一个0，容易由硬件实现，优于字符填充法）
4. 违规编码法 （只适用于采用冗余编码的特殊编码环境）

2. 差错控制

包括自动重传请求ARQ和前向纠错FEC

1.检错编码（都采用冗余编码技术）
1. 奇偶校验码 （只能检查出奇数个比特错）
2. 循环冗余码（CRC）
    - 只能做到无差错接受，凡是接收端数据链路层接受的帧都是无差错的，因为有差错的都被丢弃了

以前在CRC基础上，加上帧编号，确认重传机制，有很好的作用。现在通信线路质量提高，由于通信链路出错的概率大大降低，所以就可以区别对待

对于通信质量好的有限传输链路，数据链路层不使用确认和重传机制，即不要求数据链路层向上提供可靠传输的服务。
如果在数据链路层传输数据时出啊先了差错并且需要进行改正，那么改正差错的任务就由上层协议（如传输层TCP协议）来完成

对于通信质量较差的无线传输链路，数据链路层协议使用确认和重传机制，数据链路层向上提供可靠传输的服务

以上区别对待，可以提高通信效率
2. 纠错编码
海明码

3. 点对点协议PPP（仅支持点对点链路，只支持全双工链路）

1. PPP协议需满足的需求

1. 简单（不需要纠错，不需要序号，不需要流量控制）
2. 封装成帧
3. 透明性
4. 多种网络层协议
5. 多种类型链路
6. 差错检测（丢弃错的帧）
7. 检测连接状态（检测链路是否处于正常工作）
8. 最大传送单元（数据部分的最大长度）
9. 网络层地址协商
10. 数据压缩协商

2. PPP协议组成

1. 将IP数据报封装到串行链路的方法
2. 用来建立、配置和测试数据链路连接的链路控制协议LCP
3. 网络控制协议NCP

3. PPP协议的帧格式

4. 注意

1. PPP异步传输时，字节填充法
2. PPP在SONET/SDH等同步线路时，采用硬件完成比特填充法
3. PPP两端可以运行不同的网络层协议，但仍然可以使用同一个PPP通信

4. 局域网的数据链路层

1. 局域网特点：

1. 为一个单位所拥有，且地理范围和站点数目均有限
2. 所有站点共享较高的总带宽（即较高的数据传输率）
3. 较低的时延和较低的误码率
4. 各站为平等关系而非主从关系
5. 能进行广播和组播

2. 局域网的特性由三个要素决定： 谈判解决、传输介质、介质访问控制方式

3. 局域网有主要优点：

1. 具有广播功能，从一个站点可很方便地访问全网。局域网上的主机可共享连接在局域网上的各种硬件和软件资源
2. 便于系统的扩展和逐渐演变，个设备的位置可灵活调整和改变
3. 提高系统的可靠性，可用性和生存性

4. 介质访问控制

1. 静态划分信道 （代价较高，不适合局域网使用）
- 频分复用（FDM）
- 时分复用（TDM）
- 波分复用（WDM）
- 码分复用（CDM）——码分多址
2. 动态媒体介入控制

电磁波在1km电缆的传播时延约为5μs

- 随机接入（争用型协议）
    = ALOHA协议
    = CSMA协议
    = CSMA/CD协议（载波监听多点接入/碰撞检测）
        —— 曼彻斯特编码
        —— 最长多久知道发生碰撞 2τ （两倍的总线端到端的传播时延）—— 争用期
        —— 经过争用期还没有检测到碰撞，才能肯定这次发送不会发送碰撞
        —— 不可能进行全双工通信，只能半双工通信
        —— 截断二进制指数退避算法来确定碰撞后的重传时机（当重传16次不成功，就丢弃该帧）
        —— 以太网规定最短帧长64字节（512 bit），凡长度小于64字节的帧都是由于冲突而异常中止的无效帧，以太网争用期51.2μs
        —— 以太网规定帧间最小间隔9.6μs
        —— 信道利用率
    = CSMA/CA协议
- 受控接入

5. 以太网的MAC层

1. MAC帧的格式（DIX Ethernet V2标准 和 IEEE 802.3标准）

1. 数据字段长度小于46字节，MAC子层就会在数据字段后面加入填充字段
2. 从MAC子层向下传到物理层还要在前面插入8字节（硬件生成），包括7个字节的前同步码（作用是使接收端的适配器在接收MAC帧能够迅速调整其时钟频率），
    帧定界符（10101011，作用于前同步码一样）
3. MAC帧的FCS检验范围不包括前同步码和帧定界符
4. 在使用SONET/SDH进行同步传输时则不需要前同步码

2. 无效MAC帧（IEEE 802.3）

1. 帧长度不是整数个字节
2. 用收到的帧检验序列FCS查出有差错
3. 收到的帧的MAC客户数据字段的长度不在46~1500字节之间（有效MAC帧长度为64~1518字节之间）

6. 扩展的以太网

1. 在物理层扩展

使用集线器来连接成覆盖更大范围的多级星形结构的以太网，主机使用光纤和一堆光纤调制解调器（进行电信号和光信号的转换）来扩展主机和集线器之间的距离

好处：
1. 能够跨冲突域通信
2. 扩大以太网覆盖的地理范围
缺点：
1. 扩大后的最大吞吐量仍然是之前单个的吞吐量，也就是说 单独两个站在通信时所传送的数据会通过所有的集线器转发，使得其他的都不能在这时通信
2. 如果不同的冲突域使用不同的以太网技术（如数据率不一样），就不能使用集线器

2. 在数据链路层扩展

使用网桥（网桥对收到的帧根据其MAC帧的目的地址进行转发和过滤），后出现了以太网交换器

以太网交换机特点：
1. 实质是多接口的网桥，全双工方式，具有并行性，即能同时连通多对接口，使多对主机能同时通信（而网桥只能一次分析和转发一个帧）
2. 相互通信的主机都是独占传输媒体，无碰撞地传输数据
3. 是即插即用设备，内部的帧交换表示通过自学习算法自动地逐渐建立起来，它使用了专门的交换结构芯片，用硬件转发，其转发速率要比使用软件 转发的网桥快很多

为了增加网络的可靠性，在使用以太网交换机组网时，往往会增加一些冗余的链路，这种情况下可能就会导致以太帧在网络的某个环路中无限地兜圈子

——为了解决这种兜圈子的问题，IEEE的802.1D标准制定了生成树协议STP，不改变网络的实际拓扑，但在逻辑上切断某些链路，使得无环路的树状结构

7. 高速以太网

1. 100BASE-T以太网

在双绞线上传送100Mbit/s基带信号的星型拓扑网，使用IEEE 802.3 的CSMA/CD协议
