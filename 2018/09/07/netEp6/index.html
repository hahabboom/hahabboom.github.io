1. 域名系统DNS

  1. 顶级域名分类：

    1. 国家顶级域名：cn，us，uk等等
    2. 通用顶级域名：com，net，org，int，edu，gov，mil，aero，asia，org等等
    3. 基础结构域名（反向域名）：arpa

  2. 二级域名分类：

    1. 类别域名：ac，com，edu，gov，mil，net，org
    2. 行政区域名：适用于我国的各省，自治区，直辖市

  3. 域名服务器分类：

    1. 根域名服务器：根域名服务器是最高层次的域名服务器，也是最重要的域名服务器
      所有的根域名服务器都知道所有的顶级域名服务器的域名和IP地址
    2. 顶级域名服务器：
    3. 权限域名服务器：负责一个区的域名服务器
    4. 本地域名服务器：不属于域名服务器层次结构

为了提高域名服务器的可靠性，域名服务器吧数据复制到几个域名服务器来保存，其中一个是主域名服务器，其他是辅助域名服务器
当主域名服务器出故障时，辅助域名服务器可以保证DNS的查询工作不会中断。
主域名服务器定期将数据复制到辅助域名服务器中，而更改数据只能在主域名服务器中进行。

  1. 域名解析：
    1. 递归查询
    2. 迭代查询

2. 文件传送协议

  1. FTP基本工作原理：
    客户服务器方式
    一个FTP服务器可以同时为多个客户进程提供服务
    FTP服务器进程有两个部分组成：一个主进程（负责接收新的请求）；另外若干个从属进程（负责处理单个请求）

  2. 简单文件传送协议TFTP（很像停止等待协议）
    优点：

  3. TFTP可用于UDP环境

  4. TFTP代码所占的内存较小

主要特点：

  1. 每次传送的数据报中有512字节的数据，但最后一次可不足512字节
  2. 数据报文按序编号，从1开始
  3. 支持ASCII码或二进制传送
  4. 可对文件进行读和写
  5. 使用很简单的首部

3. 万维网WWW

超文本传送协议HTTP

请求一个万维网文档所需时间是改文档的传输时间 + 两倍RTT（一个RTT用于TCP连接，一个用于请求和接收万维网文档）

HTTP/1.0 缺点：
每请求一个文档需要两倍RTT的开销，另一种开销是万维网客户和服务器每一次建立新的TCP连接都要分配缓存和变量，
特别是万维网服务器往往要服务于大量客户的请求

HTTP/1.1使用了持续连接，可以使万维网服务器在发送响应后的一段时间内保持这条连接

HTTP/1.1 工作方式：

  1. 非流水线方式：客户收到前一个响应后才能发出下一个请求，节省建立TCP连接所需的一个RTT时间
    缺点：服务器发送完一个对象后，其TCP连接就处于空闲状态，浪费服务器资源
  2. 流水线方式：客户方位所有的对象只需环肥一个RTT时间，流水线工作方式使TCP连接中的空闲时间减少，提高下载文档效率

代理服务器（万维网高速缓存）：是一种网络实体，代理服务器把最近的一些请求和响应暂存在本地磁盘中。当新请求到达，若代理服务器发现这个请求跟暂存的请求相同，
就返回暂存的响应，而不需要按URL的地址再去互联网访问资源。

HTTP的报文结构：

HTTP状态码：

1xx —— 通知信息，如请求收到了或者正在进行处理
2xx —— 成功，如接受或知道了
3xx —— 重定向，如要完成请求还必须采取进一步的行动
4xx —— 客户的差错，如请求中有错误语法或者不能完成
5xx —— 服务器错误，服务器失效无法完后才能请求

4. 电子邮件

1. 简单邮件传送协议SMTP

由于SMTP使用客户服务器方式，因此负责发送邮件的SMTP进程就是SMTP客户，而负责接收邮件的SMTP进程就是SMTP服务器

发件人的邮件发送到发送方邮件服务器的邮件缓存后，SMTP客户就每隔一段时间对邮件缓存扫描一次，如有，就使用SSMTP的25端口
与接收方邮件服务器的SMTP建立TCP连接

SMTP不使用中间的邮件服务器

缺点：

  1. 不能传送可执行文件或其他的二进制对象。
  2. 仅限传送7位的ASCII码
  3. SMTP服务器会拒绝超过一定长度的邮件
  4. 某些SMTP的实现并没有完全按照SMTP的互联网标准

2. 邮件读取协议POP3和IMAP

POP3只要用户从POP3服务器读取了邮件，POP3服务器就把该邮件删除，后来为了解决这个问题，POP3进行了功能扩充，
让用户能够实现设置邮件读取后仍然在POP3服务器中的存放时间

IMAP最大的好处是用户可以在不同的地方使用不同的计算机，缺点是如果用户没有将邮件复制到自己的计算机上面，
邮件就会一直存放在IMAP服务器上
